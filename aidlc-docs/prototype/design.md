# Prototype Design

## エンティティ（最小限）

| エンティティ | 主要な属性 |
|-------------|-----------|
| Task | id, title, completed |

createdAt はプロトタイプでは省略。

## ユースケース（主要フロー）

### UC-1: タスクを追加する

1. テキスト入力欄にタスク名を入力
2. Enter キーまたは追加ボタンをクリック
3. タスクが未完了一覧に表示される

### UC-2: タスクを完了にする

1. タスクのチェックボックスをクリック
2. タスクが未完了一覧から消え、完了済み一覧に移動する

### UC-3: タスクを削除する

1. タスクの削除ボタンをクリック
2. タスクが一覧から消える

## モックデータ

```json
[
  { "id": "1", "title": "牛乳を買う", "completed": false },
  { "id": "2", "title": "レポートを書く", "completed": false },
  { "id": "3", "title": "メールを返す", "completed": false },
  { "id": "4", "title": "部屋を掃除する", "completed": true },
  { "id": "5", "title": "本を返す", "completed": true }
]
```

## プロトタイプのゴール

- [ ] タスクの追加・完了・削除の基本フローが動作する
- [ ] 1画面でシンプルに操作できることを確認する
- [ ] localStorage によるデータ永続化が動作する

## 技術スタック（プロトタイプ用）

| レイヤー | 技術 |
|---------|------|
| フロントエンド | React + Vite |
| スタイリング | Tailwind CSS |
| パッケージマネージャー | Bun |
| データ | localStorage |

## 対象外（プロトタイプでは作らない）

- テストコード
- エラーハンドリング
- アクセシビリティ対応
- レスポンシブデザインの細かい調整
- 型の厳密な定義
